```python
def build_claim_text(
    requirement_text: str,
    evidence_lines: list[str],
    llm_claim: str | None = None,
    edited_claim: str | None = None,
) -> str:
    """Build claim text with priority: user edit > LLM claim > template fallback.

    Args:
        requirement_text: The requirement being matched.
        evidence_lines: Evidence text lines supporting the claim.
        llm_claim: Claim text generated by the LLM (from PipelineState.proposed_claims).
        edited_claim: User-provided edited claim text (from Edited Claim: field).
    """
    if edited_claim and edited_claim.strip():
        return edited_claim.strip()
    if llm_claim and llm_claim.strip():
        return llm_claim.strip()
    if evidence_lines:
        return f"{requirement_text} Evidence includes: {evidence_lines[0]}"
    return f"No direct evidence available yet for: {requirement_text}"
```

This is the key quality fix — the old `_propose_claim_text` always used the template, ignoring LLM output. The new version has a clear priority chain: user edit > LLM claim > template fallback.
