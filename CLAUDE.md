# CLAUDE.md — Unified Automation Project

This file provides guidance for development in the `unified-automation` worktree.

## Core Commands

```bash
# Unified CLI Entry Point
python -m src.automation.main scrape --source <portal> --limit <count>
python -m src.automation.main apply --source <portal> --job-id <id> --cv <path> --dry-run

# Run All Tests
python -m pytest tests/unit/automation/ -q

# Run Specific Test File
python -m pytest tests/unit/automation/ariadne/test_c4ai_compiler.py -v
```

## Architecture

This project uses the **Ariadne Semantic Layer** to unify all browser automation.

- **`src/automation/ariadne/`**: The core "Brain" of the system.
    - `models.py`: Unified models for States, Tasks, and Paths.
    - `compiler/`: Logic to translate high-level Ariadne Paths into low-level motor scripts.
- **`src/automation/portals/`**: The "Map" of the portal.
    - `maps/`: Contains single JSON maps per portal flow (AriadnePortalMap).
- **`src/automation/motors/`**: The "Engines" that execute actions.
    - `crawl4ai/`: Uses C4A-Scripts generated by the compiler.
    - `browseros/`: (Under migration) Uses direct tool calls and fuzzy matching.

## Development Principles

1.  **Ariadne First**: Every new portal or flow MUST start as an `AriadnePortalMap` in JSON. Never hardcode logic into a motor-specific adapter.
2.  **Backend Neutrality**: Actions must be defined as `AriadneIntent` (CLICK, FILL, UPLOAD) rather than backend-specific tool calls.
3.  **States & Transitions**: Use `AriadneState` to define logical rooms (e.g., "Resume Modal"). This allows the motors to hunt for their location if the flow deviates.
4.  **Semantic Targets**: Every `AriadneTarget` should contain both `css` (for Crawl4AI) and `text` (for BrowserOS).
